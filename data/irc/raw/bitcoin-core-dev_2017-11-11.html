01:36 < sipa> BDB

01:36 < sipa> and the wallet still uses BDB
01:37 < ZSky> why no BDB anymore?
01:38 < sipa> many many reports of corruption
01:38 < sipa> incompatibilities between versions (you can't downgrade, and in some cases you can't even upgrade)
01:39 < sipa> and the fact that BDB was indirectly the cause for one of the largest operational problems the system has seen in years (read BIP50)
01:39 < ZSky> oh really?

01:40 < ZSky> tl;dr what was it?
01:40 < sipa> while we don't need any locks at all
01:40 < sipa> you need to configure BDB with "i need up to X locks", and if any atomic database operation touches more internal records than you have configured locks, the operations fails
01:41 < sipa> this is because BDB is a multi-application database system (multiple processes can simultaneously open and modify the database, and there is consistency across them), which is something we don't care about at all
