08:29 < PaulTroon> I was also trying to understand that BIP30 attack scenario, snot sure if I understand it.. a coinbase_1 with an unspent txout to A, and then create a duplicate coinbase_2 that has a txout to B ?
08:30 < PaulTroon> So then a tx that spends the txout to A is invalid because coinbase_2 does not have that txout A ?
08:30 < sipa> PaulTroon: coinbase_1 is created, the chain forks into branch A and branch B (just a 1 or 2 block reorg, otherwise harmless); in branch A, coinbase_2 overwrites coinbase_1; in branch B this does not happen
08:30 < sipa> branch B eventually wins
08:31 < sipa> nodes that have seen branch A will have overwritten coinbase_1, and then _deleted_ it when switching to branch B (as they're "undoing" the creation of coinbase_2, having forgotten something was overwritten)
08:31 < sipa> nodes that have not seen branch A will think coinbase_1 still exists
08:32 < sipa> if now someone spends the coinbase_1 output, it permanently forks the network between those that have seen branch A and those who have not
