17:03 <mooncoder> is the hdseedid value sufficient to recover a bitcoin-core wallet?
17:03 <sipa> no
17:05 <mooncoder> Wallets aren't deterministic?
17:05 jespada has quit [Quit: Textual IRC Client: www.textualapp.com]
17:05 <mooncoder> so you need to make regular backups , not just once?
17:05 <sipa> It's complicated. The keys are deterministic, but the hdseedid value is just a hash of the master key.
17:06 <sipa> A backup once suffices, unless you import keys, to recover your funds.
17:06 <mooncoder> Ok
17:06 <sipa> At least if the wallet was created using Bitcoin Core 0.13 or later.
17:07 <sipa> Before that, the keys were not deterministic.
17:08 dviola has joined #bitcoin-core-dev
17:15 <mooncoder> Cool
17:16 <mooncoder> Why doesn't bitcoin-core just export the master key? it would make offline backup easier vs a dat file with custom binary format
17:23 <sipa> Being a full node wallet, which doesn't rely on trusted third party services, makes recovering a wallet generally an expensive operation.
17:24 <sipa> And a seed doesn't let you recover a whole wallet anyway, just the keys/funds. It can't help you get information about invoices back, e.g. if you assign labels to addresses to figure out who paid you.
17:24 <sipa> So it follows a model where a wallet is a file, and you need to back up that file.
17:24 <sipa> Part of that is just historical - the Bitcoin Core wallet predates all other wallet software, and still has a design that dates back to this time (mostly).
17:25 <sipa> But with the more recent descriptor wallets you can export the private descriptors, which suffices to recover funds.
17:26 <sipa> It's a bit longer than just a master key or seed phrase, but encompasses a lot more (e.g. it's compatible with multisig setups, different script types/versions, ...).
17:29 <mooncoder> Interesting
17:29 <sipa> With the older wallets it's also possible to dump the master key if you really want to (using dumpwallet RPC) which should be enough to recover the keys, but comes with caveats.
17:29 <sipa> I only said that hdseedid isn't enough.
17:29 <mooncoder> Ah ok
17:33 <mooncoder> I'm surprised that bitcoin-core still doesn't support passphrases, was expecting to support it by now
17:34 <sipa> It supports encrypting the wallet with a passphrase since v0.5
