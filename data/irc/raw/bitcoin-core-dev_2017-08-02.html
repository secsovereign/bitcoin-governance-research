21:59 < gmaxwell> jnewbery: https://twitter.com/jfnewbery/status/892856516985991170 I think this is misleading. Retarget modified the difficulty from the prior block, so it's likely to double apply a dip, not undo it.
22:00 < gmaxwell> Though it depends on when in the retarget window the crazy-change applies.
22:00 < gmaxwell> E.g. if 20 blocks before the retarget the 12h rule triggers, it will then drop the difficulty 73.7% (because when it triggers it triggers 6 times in a row...)
22:01 < gmaxwell> then you'll get to the retarget, and the code will go "oh wow, took longer than expected for the last 2016 blocks, better lower the difficulty".
22:11 < gmaxwell> E.g. the worst case is probably something like you run at 1/11th the hashrate for a whole retarget interval, then just before the end, hit the magic rule. This drops your difficulty by 3.8, then the retarget will apply and drop the diffuclty by a factor of 11, resuting in blocks being about 4x too fast.
